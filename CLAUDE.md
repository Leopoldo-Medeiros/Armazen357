This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Stack

- **Laravel 11** (PHP 8.2+) with **Laravel Breeze** for authentication scaffolding
- **Frontend**: Vite + Tailwind CSS + Alpine.js
- **Local environment**: Lando (Docker) with PHP 8.2, MySQL 8.0, nginx
- **Database**: MySQL via Lando in dev; SQLite used in tests

## Development

Start all services (server, queue, logs, Vite) simultaneously:
```bash
composer run dev
```

Or run individually:
```bash
php artisan serve
npm run dev
```

Build frontend assets:
```bash
npm run build
```

## Lando (local environment)

```bash
lando start         # start containers
lando install       # full setup: composer, .env, migrate+seed, npm install, storage:link
lando migrate       # run migrations
lando rollback      # rollback last migration
lando tinker        # open Tinker shell
```

## Testing

Tests use PHPUnit and run against the configured database (not SQLite in-memory by default — check `phpunit.xml` to toggle).

```bash
php artisan test                          # run all tests
php artisan test --filter TestName        # run a specific test
php artisan test tests/Feature/ProfileTest.php  # run a specific file
```

## Architecture

**Authentication**: Provided by Laravel Breeze. Auth controllers live in `app/Http/Controllers/Auth/`, routes in `routes/auth.php`. Do not hand-roll auth logic.

**Layouts**: Two Blade component layouts:
- `app/View/Components/AppLayout.php` → `resources/views/layouts/app.blade.php` — authenticated pages
- `app/View/Components/GuestLayout.php` → `resources/views/layouts/guest.blade.php` — public/auth pages

**Blade components**: Reusable UI components (buttons, inputs, modals, dropdowns, nav-links) in `resources/views/components/`.

**Domain**: Currently early-stage. A `menu_items` table (name only) exists alongside the standard Breeze user/session/cache/jobs tables.

**Routes**: `routes/web.php` handles app routes (dashboard requires auth+verified); `routes/auth.php` (generated by Breeze) handles auth flows.

**Form Requests**: Validation lives in `app/Http/Requests/` — extend this for new form validation rather than validating in controllers.
